# AI Processing Design

## Purpose

The AI processing subsystem scores CVs and generates structured profile data from uploaded files.

## Main Capabilities

- Determine whether an uploaded document is a CV
- Produce structured scoring output with component scores
- Produce explanatory highlights, strengths, and weaknesses
- Generate a LinkedIn-style structured profile JSON
- Support advanced asynchronous review for longer-running scoring

## Processing Stages

1. User uploads file
2. Backend validates extension and size
3. Backend extracts text from `PDF`, `DOCX`, or `DOC`
4. Backend rejects empty or unreadable documents
5. Backend sends extracted text and scoring rules to OpenAI
6. Backend validates the returned JSON structure
7. Backend computes and clamps the overall score
8. Backend optionally persists score data or profile JSON
9. Backend returns result to frontend

## Scoring Model

- Component categories currently include:
- education
- experience
- skills
- ai_signal
- projects

- A weighted overall score is computed from those components
- Confidence is generated by the AI response
- There is support for escalation to a stronger model when confidence is low

## Async Advanced Review

- Advanced review creates an in-memory task record with a process id
- The frontend polls task status
- The frontend retrieves final results after status becomes `done`
- Current architecture is suitable for small-scale deployment but not yet for durable distributed processing

## Profile Generation Model

- The system also generates a structured candidate profile
- The generated structure includes basics, experience, education, projects, skills, and other optional resume sections
- The generated profile payload is stored under `profile.profile_json`

## Risks

- AI output can vary by model and prompt behavior
- Weak or poorly structured CVs may produce lower-confidence extractions
- Scanned CVs without OCR may fail before AI processing
- Async jobs are not persisted across server restarts
